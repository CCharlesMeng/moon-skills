# 评测用例与回归验证

本文件是 refine-skill 第五步的机械细节:evals 文件格式、维护规则、重放与打分的子代理提示词模板。只在执行验证时加载。

## evals/evals.json 格式

沿用 Anthropic skill-creator 约定,放在目标 skill 目录内,随 skill 一起携带——带着测试集的 skill 才是可信的 skill。

```json
{
  "skill_name": "<目标 skill 名>",
  "evals": [
    {
      "id": 1,
      "name": "short-descriptive-name",
      "prompt": "触发情境:用户的原始请求或最小复现指令",
      "expected_output": "成功长什么样,一句人话",
      "expectations": [
        "可逐条判定的行为断言,如:输出包含 X",
        "过程断言,如:重活委派给了子代理而非主 agent 亲自做"
      ],
      "note": "可选:偶然错误标记为观察中;复发后删除此标记升级为系统性"
    }
  ]
}
```

## 维护规则

- 每次复盘追加:本次系统性错误的用例 + 1-2 个本次做对了的行为(防退化)。
- 偶然错误也追加,`note` 标记观察;复发时升级为正式用例。
- 只写行为断言,不写实现细节——断言应在下次改写后依然成立。
- 修剪已失效的情境,总量控制在约 10 条代表性用例,防止测试集自己发福。
- 好断言有区分度:skill 真做对才过,做错必挂。"文件存在但内容为空"这类表面达标要判挂。

## 重放子代理提示词模板(executor)

```
你要执行一个 skill。除本提示外你没有任何背景,不要假设任何未提供的信息。

【输入材料】
- skill 正文:<改写后的 SKILL.md 全文;若情境涉及 references,一并附上>
- 情境:<用例的 prompt>

【输出格式】
- 按 skill 指示处理该情境,输出你实际会执行的动作序列与最终产出。

【完成判据】
- 输出足够具体,能被逐条对照断言判定;不省略中间决策。
```

## 打分子代理提示词模板(grader)

```
你是评测打分者。除本提示外你没有任何背景。

【输入材料】
- 断言列表:<该用例的 expectations>
- 执行记录:<executor 的完整输出>

【输出格式】
- 每条断言:pass / fail + 证据原文引用。无部分得分。
- 末尾:pass/fail 计数;以及你发现的断言缺陷——永远会过的断言、
  没被任何断言覆盖的重要行为。没有就写"无"。

【完成判据】
- 每条断言都有判定和引用证据;表面达标(如仅名称正确、内容空洞)判 fail。
```

## 通过标准

- 新增错误用例:每例跑约 3 次取多数,不再复发。
- 存量用例:全过。任何退化 → 改写过头(通常收太紧),回退再验。
- 用实际使用档位的模型跑重放,而不是当前会话的模型。
